# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in DC Tools, please report it responsibly:

1. **DO NOT** create a public GitHub issue
2. Email us at: **security@deniscasian.com**
3. Include:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Any suggested fixes (optional)

## Response Time

We aim to respond to security reports within:
- **Initial Response:** 24-48 hours
- **Resolution:** Depends on severity (critical issues prioritized)

## Security Measures

DC Tools implements the following security measures:

- ✅ All data processing happens client-side
- ✅ No user data is stored on servers
- ✅ HTTPS enforced on all connections
- ✅ No third-party tracking scripts
- ✅ Regular security audits

## Scope

This security policy applies to:
- https://tools.deniscasian.com
- Associated subdomains (qr.deniscasian.com, short.deniscasian.com)

Thank you for helping keep DC Tools secure!
